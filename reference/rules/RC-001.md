# RC-001 — Idle EC2 Instance

## Summary
| Field | Value |
|---|---|
| Rule ID | RC-001 |
| Provider | AWS |
| Resource type | EC2 instance (`*` — any type) |
| Severity | medium |
| Category | idle |
| Certified | Yes (Batch 1) |
| File | `idle_ec2.go` |

## Description

Flag EC2 instances that have been stopped for 30+ days. The instance itself is no longer billing for compute, but attached EBS volumes and snapshots continue to incur cost.

## Data needed from Discoverer

- `resource.Status: statusStopped`
- State history: 30+ days in stopped state

## Thresholds

- `idleThresholdDays = 30`

## Cost calculation

PricingAware — uses per-resource monthly cost from Aggregator. Savings = 100% of current cost (since termination eliminates the EBS and snapshot charges that continue to bill).

## Remediation

1. Verify no workload depends on this instance
2. Create an AMI backup if needed (snapshot first)
3. Terminate the instance via EC2 Console (or one-click via ZopNight if auto-rem enabled)
4. Delete associated EBS volumes if no longer needed

## Console URL pattern

```
https://%s.console.aws.amazon.com/ec2/home?region=%s#InstanceDetails:instanceId=%s
```

## Auto-remediation

Yes — certified. Apply Remediate button visible on the card.

Workflow:
1. Precondition: confirm still stopped
2. Action: snapshot EBS, terminate instance, delete EBS volumes
3. Validate: confirm termination
4. Notify: success or failure
